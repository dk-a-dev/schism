# Schism v0.2 — Change Plan (backend + app)

Compiled from feedback on 2026-07-06. Grouped by **App (Android)** and **Backend (Go)**, with
priorities and the two decisions that gate the big items.

---

## A. App (Android)

### A1. Offline-first everything  *(biggest; architectural)*
Goal: add / edit / split / view **works with no network**, and pending changes sync automatically
when the network or backend comes back.
- **Cache everything in Room**: group details, participants, expenses, balances, activity (today only
  some are cached, so an in-group screen can't load offline). Repos read Room first, refresh in bg.
- **Write outbox**: every mutation (create/update/delete expense, create/edit group, settle,
  push-to-split) is written to Room immediately (optimistic), enqueued in an `outbox` table with an
  idempotency key, and flushed by a WorkManager sync job when connectivity returns.
- **Conflict handling**: last-write-wins per resource; idempotency keys prevent double-post.
- **Connectivity/health indicator**: a **cloud-with-slash** icon in the top bar when offline or the
  backend is unreachable; auto-clears and syncs on reconnect. A "pending sync" count somewhere.

### A2. Auth — login / logout / session  *(needs decision D1)*
- Login + Register screens (separate from the first-run walkthrough).
- Persist the session token (already in DataStore); auth-gate the app on launch.
- **Logout**: clear token + local caches, return to login.

### A3. Inbox / SMS
- **Keep-personal must not delete** the message — mark it `PERSONAL`; add **filter chips**
  (All / To split / Personal / Added-to-group).
- **Inline edit** of a transaction in the list: merchant/title, amount, direction (sent vs received),
  date — because SMS parsing isn't always right.

### A4. Split flow
- Inbox **"Split" opens the full expense editor** (all split modes + participant selection),
  prefilled from the transaction (title/amount/date/payer) and group-preselected — **not** the
  lightweight PushToSplit page. Retire/repurpose PushToSplit into just a group-pick step that feeds
  the editor.

### A5. Bottom nav
- **Single-line labels** (Dashboard currently wraps and breaks the bar). Possibly shorten labels.

### A6. On-device AI  *(needs decision D2 — the 401/403)*
- Make download **just work**: default to an **ungated** model (no token) or one **we host**; keep
  "bring your own model + token" as an option (model picker: Recommended / Custom).
- Clearer errors (401 = license not accepted / bad token).

### A7. Settings
- **Collapsible/expandable sections** (accordion) now that there are many.
- **Logout** entry.
- **Data**: Reset (local wipe) + **Delete account** (calls backend).

### A8. Activity
- **Filter chips** (All / Added / Updated / Removed / Settled / by person).

## B. Backend (Go)

### B1. Auth  *(needs D1)*
- `users`: add `email` (unique) + credential (password hash, or phone+OTP).
- Endpoints: `POST /v1/auth/register`, `POST /v1/auth/login` → token, `POST /v1/auth/logout` → revoke.
- Keep the existing hashed bearer-token table for sessions.

### B2. Account
- `DELETE /v1/users/me` — delete the account and detach/remove owned data.

### B3. Offline-sync support
- Ensure **update & delete are idempotent** (create already is). Optionally add a lightweight
  **pull-changes** endpoint (`GET /v1/groups/{id}/changes?since=…`) or keep per-resource refresh.

### B4. Seed / demo data
- A **seed command** (`make seed` or a one-off): wipe + insert **2–4 mock groups**, mock people, and
  mock expenses/transactions for a clean demo restart.

### B5. Misc
- Activity filtering stays **client-side** (activities are already per-group) — no endpoint change.

## Priorities

- **P0 (quick, do first):** A5 bottom bar · A6 AI ungated model · B4 seed + DB wipe · A4 split→editor · A3 inbox filters + inline edit.
- **P1:** A1 offline-first (Room caching + outbox + indicator) · B3 sync support.
- **P2:** A2 + B1 auth (login/logout/session) · B2 account delete.
- **P3:** A7 settings accordion · A8/A3 filters polish.

## Decisions needed
